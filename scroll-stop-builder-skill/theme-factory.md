## How to Apply a Theme

1. **Show the showcase** — Present theme options visually if the user is unsure
2. **User selects** — Either from the 10 pre-built or requests a custom theme
3. **Read theme specs** — Load the full color and font specification
4. **Apply consistently** — Every element in the artifact uses theme colors and fonts
5. **Verify readability** — Confirm contrast ratios meet accessibility standards

---

## Custom Theme Generation

When the user wants a custom theme based on their brand:

1. Ask for: primary brand color, secondary color, any existing fonts, mood/personality
2. Build a 4-color palette: background, surface, text, accent
3. Select font pairing from Google Fonts that matches the mood
4. Document the custom theme in the same format as the pre-built themes above

---

## Application Rules

- **Consistency**: Every heading, body text, and color element in the artifact must use the theme — no mixing
- **Hierarchy**: Dark themes → light text; light themes → dark text. Never fight the background
- **Accent scarcity**: The accent color appears on CTAs and highlights only — overuse kills its impact
- **Font loading**: Always include Google Fonts import URL or specify system font fallbacks
